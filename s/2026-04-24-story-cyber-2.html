Chinese APT Weaponizes Outlook, Slack, and Discord as C2 Channels to Spy on Mongolia

High ⚠ Mitigation Only

A Chinese state-sponsored threat actor is weaponizing legitimate cloud services — Microsoft Outlook, Slack, Discord, and file.io — as command-and-control (C2) infrastructure to conduct espionage operations against Mongolian government targets. By hiding C2 traffic inside trusted platforms, the adversary makes detection extraordinarily difficult since malicious commands are nearly indistinguishable from normal business traffic. The multi-channel approach provides built-in redundancy, ensuring persistent access even if one service is blocked.

📅 Apr 23, 2026 9:00 PM EDT| Source: darkreading ↗
▶Attack Flow
Chinese APT Abuses Outlook, Slack, Discord, and file.io as C2 Channels Targeting Mongolia
The APT programs custom malware to use legitimate cloud services as dead drops: it polls Outlook mailboxes (likely via Microsoft Graph API), Slack channels, and Discord (via webhooks or bot tokens) for encoded commands, executes them on the victim endpoint, and exfiltrates results back through these same channels or via the ephemeral file-sharing service file.io, which auto-deletes files after a single download to destroy forensic evidence.

❓Am I Affected?

You are likely at risk if any of the following apply:

⚠Your organization operates in Mongolia or has diplomatic, governmental, or NGO ties to Mongolian entities
⚠Your environment allows unrestricted outbound HTTPS access to Outlook/Microsoft 365, Slack, and Discord endpoints
⚠Your organization does not inspect or monitor API-level usage of Microsoft Graph, Slack, or Discord for anomalous behavior
⚠Your environment permits unmanaged OAuth app registrations or third-party integrations in Microsoft 365 or Slack without review
✓Your organization has no business use for Discord and has already blocked Discord domains at the network perimeter
✓Your environment enforces application allowlisting and monitors all outbound cloud service connections via a CASB or SASE solution
🛠Fix & Recommendations
⚠ Immediate 24–72 hours
  • 1Block outbound access to file.io (file.io, *.file.io) at your web proxy and firewall — this ephemeral sharing service has minimal legitimate enterprise use and is commonly abused for exfiltration.
  • 2Block or restrict Discord (discord.com, cdn.discordapp.com, gateway.discord.gg) at the network perimeter if Discord is not a sanctioned business tool in your organization.
  • 3Audit Microsoft 365 tenant for unauthorized OAuth app registrations and review Graph API access logs (via Azure AD sign-in logs and unified audit log) for unusual mailbox access patterns such as automated reading/writing of draft emails.
  • 4Review Slack workspace integrations, bot tokens, and incoming/outgoing webhooks for any unauthorized or unrecognized entries; revoke suspicious tokens immediately.
  • 5Deploy or update endpoint detection rules to flag processes making high-frequency HTTPS requests to Outlook, Slack, or Discord APIs from non-browser, non-standard user-agent strings.
● Strategic This week & beyond
  • 1Deploy a Cloud Access Security Broker (CASB) or Secure Access Service Edge (SASE) solution to gain visibility into and control over cloud service API usage, enabling detection of anomalous patterns like bot-like polling intervals or unusual data volumes in email drafts.
  • 2Implement behavioral analytics and UEBA (User and Entity Behavior Analytics) to detect Living-off-Trusted-Services techniques — baseline normal Graph API, Slack API, and Discord API usage patterns and alert on deviations such as automated mailbox manipulation or unusual channel activity.
  • 3Enforce strict OAuth app governance in Microsoft 365 and Slack: require admin consent for all third-party app registrations, implement app allowlisting, and regularly audit granted permissions.
  • 4Adopt a Zero Trust network model with microsegmentation that treats all outbound cloud traffic as potentially adversarial, requiring continuous authentication and authorization verification rather than trusting traffic based solely on destination domain reputation.
  • 5Establish a threat intelligence sharing relationship with peer organizations and national CERTs (especially if operating in the Central/East Asian region) to receive early indicators of Chinese APT targeting campaigns.
📚Concept BriefingClick a tag to expand
Living-off-Trusted-Services
An adversary technique where attackers use legitimate cloud platforms (email, chat, file sharing) as command-and-control infrastructure instead of dedicated malicious servers, making their traffic blend in with normal business communications.
This Chinese APT built its entire C2 infrastructure on Outlook, Slack, Discord, and file.io — services that firewalls and blocklists inherently trust — rendering traditional perimeter defenses ineffective.
Dead Drop C2
A communication method where the attacker and malware exchange messages indirectly through a shared location (like an email draft or chat message) rather than communicating directly, borrowing the concept from traditional espionage tradecraft.
The APT's malware checks Outlook drafts and chat messages for encoded commands and posts results back to the same platforms, using them as digital dead drops that leave minimal direct connection between attacker and victim.
C2 Channel Redundancy
The practice of establishing multiple independent command-and-control pathways so that if one is disrupted, others remain operational, ensuring persistent access to compromised systems.
By simultaneously leveraging four different cloud services, this APT ensured that blocking any single channel would not sever their access — a self-healing C2 architecture built from ubiquitous SaaS products.
Ephemeral File Staging
Using temporary or self-destructing file-sharing services to transfer stolen data, where files are automatically deleted after download, destroying forensic evidence.
The APT used file.io's single-download auto-deletion feature to exfiltrate data from Mongolian targets while minimizing the forensic footprint left behind.
State-Sponsored Espionage
Cyber operations conducted by or on behalf of a nation-state government, typically targeting foreign governments, militaries, or critical infrastructure for intelligence collection rather than financial gain.
This campaign is attributed to a Chinese state-sponsored threat actor conducting intelligence-gathering operations against Mongolian government targets, consistent with China's geopolitical interests in the region.
🔍Threat Hunting Signals
High Anomalous Microsoft Graph API mailbox access patterns ►
Look for non-interactive or automated access to Outlook mailboxes — specifically, repeated read/write operations on draft emails at regular intervals (e.g., every 30-60 seconds) from non-standard client applications. This indicates malware polling an email draft folder as a C2 dead drop. Check for unusual OAuth tokens or application IDs accessing the mailbox.
Microsoft 365 Unified Audit LogAzure AD Sign-in LogsMicrosoft Graph API activity logs
High Non-browser processes connecting to Discord or Slack APIs ►
Identify endpoint processes (not standard browsers or official Slack/Discord desktop apps) making HTTPS connections to discord.com, cdn.discordapp.com, gateway.discord.gg, or api.slack.com. Malware using these as C2 will typically present unusual user-agent strings, connect at fixed intervals, and transfer small encoded payloads. Cross-reference with process creation logs to identify the parent executable.
EDR telemetryWeb proxy logsDNS query logsFirewall logs
High Outbound connections to file.io ►
Any outbound HTTP/HTTPS traffic to file.io or *.file.io should be investigated, especially from server or workstation endpoints where ephemeral file sharing has no business justification. Look for POST requests (file upload/staging) followed by GET requests from external IPs (attacker retrieval). The single-download deletion feature makes timing critical for forensic capture.

Medium Periodic beaconing behavior to cloud service endpoints ►
Analyze network traffic for regular, timed connections to Microsoft, Slack, or Discord endpoints that exhibit beacon-like periodicity (e.g., connections every 30, 60, or 120 seconds with low jitter). Legitimate user traffic to these services is bursty and irregular; automated C2 polling follows predictable timing patterns. Use statistical analysis on connection intervals to detect machine-driven communication.

Medium Suspicious OAuth application registrations or consent grants ►
Monitor for new OAuth application registrations in Azure AD or Slack that request mail read/write permissions, especially those created by compromised user accounts or with generic/unusual display names. Attackers may register rogue apps to obtain persistent API tokens for mailbox manipulation as part of the C2 channel setup.
