⚠ Vulnerability Advisory

MOVEit, Again: Progress Patches Critical Auth Bypass in Automation

CVSS
9.8
Critical ✓ Patch Available

Progress Software has disclosed a critical authentication bypass vulnerability in MOVEit Automation, its enterprise file-transfer orchestration platform. Because MOVEit Automation holds privileged credentials to downstream SFTP servers, cloud storage, and partner systems, a pre-auth bypass effectively hands attackers control of an organization's bulk data pipeline — the same blast radius Cl0p exploited in the 2023 MOVEit Transfer mass-exploitation campaign.

📅 May 04, 2026 8:18 AM EDT| Source: BleepingComputer ↗
▶Attack Flow
Critical Auth Bypass in Progress MOVEit Automation (May 2026)
An unauthenticated attacker with network access to a MOVEit Automation endpoint can reach privileged functionality without supplying valid credentials, due to a flaw in the application's request-handling/authentication logic. Successful exploitation yields administrative control over scheduled file-transfer tasks and the credentials they use.

❓Am I Affected?

You are likely at risk if any of the following apply:

⚠You operate Progress MOVEit Automation anywhere in your environment (on-prem or self-hosted cloud)
⚠Your MOVEit Automation administrative interface or API is reachable from the internet or untrusted network segments
⚠Your MOVEit Automation instance stores credentials for downstream SFTP, S3, Azure Blob, or partner systems
✓You have already applied the May 2026 Progress patch and rotated stored secrets
✓You only use MOVEit Transfer (the separate product) and have no MOVEit Automation deployment
⚠Your environment exposes MOVEit Automation only over a tightly restricted management VLAN with MFA-protected jump hosts
🛠Fix & Recommendations
⚠ Immediate 24–72 hours
  • 1Identify all MOVEit Automation instances (including dev/test/DR) and apply the May 2026 patched release from Progress immediately — treat as emergency change.
  • 2If patching cannot occur within hours, block external and untrusted-network access to MOVEit Automation web/API ports at the firewall/WAF until patched.
  • 3Rotate all credentials and API keys stored in MOVEit Automation tasks (SFTP, cloud storage, partner systems, service accounts).
  • 4Hunt for indicators of compromise: review authentication logs, task creation/modification history, and outbound transfers since the disclosure window and earlier.
  • 5Engage incident response on standby and snapshot affected systems for forensic preservation before patching, where feasible.
● Strategic This week & beyond
  • 1Place MOVEit Automation behind a reverse proxy or zero-trust gateway requiring MFA and source-IP allowlisting; never expose admin/API surfaces directly to the internet.
  • 2Segment the MFT platform on its own VLAN with strict egress controls; the MFT host should only initiate connections to known, allowlisted endpoints.
  • 3Adopt a secrets vault (HashiCorp Vault, CyberArk, etc.) so MFT tasks fetch short-lived credentials at runtime instead of storing long-lived secrets in the platform.
  • 4Establish a rapid-patch SLA (e.g., 24–72 hours) specifically for MFT, VPN, and edge-management products given their consistent targeting by ransomware crews.
  • 5Implement DLP and anomaly detection on MFT egress: alert on volume spikes, new destinations, or off-schedule transfers that match Cl0p-style staging behavior.
📚Concept BriefingClick a tag to expand
Authentication Bypass
A flaw where application logic allows access to protected functionality without valid credentials, typically because an auth check is missing or skippable on some code path.
The MOVEit Automation flaw lets attackers reach privileged endpoints with zero credentials — the most dangerous class of web vulnerability.
Managed File Transfer (MFT)
Enterprise software category for secure, auditable, scheduled bulk file movement between systems and organizations.
MFT platforms like MOVEit are chokepoints holding data and credentials from many downstream systems, making each vulnerability a systemic risk.
Mass Exploitation
An attack pattern in which a single vulnerability is weaponized at scale against every reachable instance, typically within days of disclosure.
Cl0p's 2023 MOVEit Transfer campaign is the canonical example; defenders must assume the same compressed timeline here.
Patch Diffing
Reverse-engineering a vendor patch by comparing pre- and post-patch binaries to locate the fixed code path and reconstruct an exploit.
Once Progress released the fix, sophisticated actors began patch diffing immediately — shrinking the safe window to hours or days.
Blast Radius
The scope of systems, data, and downstream relationships an attacker can reach from a single compromised asset.
MOVEit Automation's stored credentials and pipeline access make its blast radius enterprise-wide, including partner and cloud environments.
