UNC6692 Poses as IT Helpdesk on Microsoft Teams to Drop Custom SNOW Malware

High ⚠ Mitigation Only

A newly tracked threat group designated UNC6692 is impersonating corporate IT helpdesk personnel via Microsoft Teams to social-engineer employees into installing a custom malware suite called SNOW. The campaign exploits the implicit trust users place in collaboration platforms, bypassing traditional email-based phishing defenses entirely. Once SNOW is deployed on a victim endpoint, it provides the attacker with persistent access and command-and-control capabilities within the compromised environment.

📅 Apr 23, 2026 2:16 PM EDT| Source: The Hacker News ↗
▶Attack Flow
UNC6692 – IT Helpdesk Impersonation via Microsoft Teams Deploying SNOW Malware
UNC6692 creates or compromises accounts mimicking legitimate IT helpdesk identities and initiates Microsoft Teams chats with targets—often leveraging external tenant communication enabled by default—to guide victims into accepting remote assistance sessions or downloading the SNOW malware payload, all without exploiting any software vulnerability.

❓Am I Affected?

You are likely at risk if any of the following apply:

⚠Your organization uses Microsoft Teams with external access (federation) enabled, allowing users to receive messages from outside your tenant.
⚠Your employees are not trained to recognize social engineering attacks delivered through collaboration platforms such as Teams or Slack.
⚠Your environment lacks endpoint detection and response (EDR) capable of identifying unknown or custom malware payloads.
⚠Your IT helpdesk does not have a documented, out-of-band verification process for support requests initiated via chat.
✓Your organization has disabled external access in Microsoft Teams and only allows communication with explicitly approved tenant domains.
✓Your environment enforces application whitelisting that blocks execution of unapproved software downloaded by end users.
🛠Fix & Recommendations
⚠ Immediate 24–72 hours
  • 1Restrict Microsoft Teams external access immediately: In the Teams Admin Center, navigate to Org-wide settings > External access and set 'Users can communicate with other Teams users' to 'Block all external domains' or whitelist only trusted partner domains.
  • 2Issue an organization-wide alert to all employees warning them that IT helpdesk will never initiate support sessions via Teams chat and providing the legitimate channel (e.g., ticketing system, phone number) for IT support requests.
  • 3Hunt for indicators of the SNOW malware across endpoints by scanning for recently downloaded executables from Teams chat sessions; review EDR telemetry for unusual process trees originating from Teams-related file downloads in the past 30 days.
  • 4Audit Microsoft Teams chat logs in Microsoft 365 Unified Audit Log for external tenant conversations where the display name matches IT helpdesk naming patterns (e.g., 'IT Support', 'Helpdesk', 'IT Admin') from unfamiliar domains.
  • 5Block remote assistance tools (e.g., Quick Assist, AnyDesk, TeamViewer) on endpoints where they are not operationally required using application control policies or Group Policy.
● Strategic This week & beyond
  • 1Extend phishing awareness training to cover collaboration platforms: develop and run simulated social engineering exercises specifically targeting Teams, Slack, and other chat tools, not just email.
  • 2Implement Conditional Access policies in Azure AD/Entra ID to require multi-factor authentication and compliant devices before Teams sessions are established, reducing the value of compromised external accounts.
  • 3Deploy application whitelisting (e.g., Microsoft Defender Application Control or AppLocker) across all user endpoints to prevent execution of unapproved binaries regardless of how they are delivered.
  • 4Establish a formal, out-of-band verification protocol for all IT support interactions: any request for software installation, remote access, or credential changes must be confirmed via a second trusted channel (e.g., phone callback to a published number).
  • 5Integrate Microsoft Teams telemetry into your SIEM platform and build detection rules for anomalous external chat patterns, file transfers from external tenants, and rapid-trust social engineering indicators such as short conversations preceding file downloads.
📚Concept BriefingClick a tag to expand
Collaboration Platform Phishing
Social engineering attacks delivered through business chat tools like Microsoft Teams or Slack, rather than traditional email, exploiting the implicit trust users place in internal communication platforms.
UNC6692's entire initial access strategy relies on abusing Microsoft Teams as a trusted channel to impersonate IT staff, bypassing email-based security controls.
External Tenant Federation
A Microsoft Teams configuration that allows users in one organization to send messages to users in a completely different organization's tenant, enabled by default in many deployments.
This default-on setting is what allows UNC6692 to initiate conversations with victims without ever compromising the target organization's own infrastructure.
Custom Malware Suite
Purpose-built malicious software created by a specific threat actor, as opposed to commodity or open-source tools, making it harder to detect with signature-based defenses.
SNOW is a previously undocumented malware suite custom-built by UNC6692 for persistence and C2, meaning traditional antivirus signatures are unlikely to catch it initially.
Pretexting
A social engineering technique in which an attacker creates a fabricated scenario (the pretext) to manipulate a victim into performing an action or divulging information.
UNC6692 constructs a convincing IT helpdesk pretext—complete with professional jargon and urgency—to persuade victims to install malware they believe is a legitimate security update.
Uncategorized Threat Cluster
A designation (often prefixed 'UNC') used by threat intelligence firms for a group of related intrusion activity that has not yet been formally attributed to a named threat actor or nation-state.
UNC6692 is a newly tracked cluster, meaning its full scope, sponsorship, and long-term objectives are still being investigated by researchers.
🔍Threat Hunting Signals
High External Teams chat initiations with IT helpdesk-themed display names ►
Search Microsoft 365 Unified Audit Log and Teams message events for incoming chats from external tenants where the sender's display name contains keywords such as 'IT Support', 'Helpdesk', 'IT Admin', 'Tech Support', or 'Service Desk'. High volume of such contacts to multiple users in a short window is a strong indicator of a campaign.
Microsoft 365 Unified Audit LogMicrosoft Teams Admin Center chat logsMicrosoft Defender for Cloud Apps activity logs
High File downloads originating from Teams chat sessions ►
Monitor EDR telemetry for executable files (.exe, .msi, .ps1, .bat, .scr) written to disk from Teams-related processes (e.g., Teams.exe child processes or files in the Teams Downloads folder). Correlate with external tenant chat events to identify socially engineered payload delivery.

High Unexpected remote assistance tool execution ►
Detect launches of Quick Assist, AnyDesk, TeamViewer, ScreenConnect, or similar remote access tools on endpoints where they are not standard software. Look for process creation events with these tool names, especially when preceded by Teams activity within a short time window.

Medium Unknown or unsigned executables with persistence mechanisms post-Teams interaction ►
Hunt for newly created scheduled tasks, registry run keys, or startup folder entries that reference executables not present in your organization's software baseline, particularly on hosts where external Teams chat activity was recently logged.

Correlate network traffic logs for beaconing patterns (regular interval HTTP/HTTPS requests to uncommon or newly registered domains) from endpoints where Teams-originated file downloads were detected. The SNOW malware's C2 communication may use standard web protocols to blend in.
