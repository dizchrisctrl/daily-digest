FBI and Indonesian Police Dismantle W3LL Phishing Empire Behind $20M in Fraud

High ⚠ Mitigation Only

A joint FBI and Indonesian National Police operation has seized the infrastructure of the W3LL phishing-as-a-service platform and detained its alleged developer. W3LL operated since at least 2023 as a turnkey phishing marketplace serving over 500 threat actors, specializing in adversary-in-the-middle (AiTM) attacks that bypassed Microsoft 365 multi-factor authentication. The operation is linked to an estimated $20 million in fraud attempts and thousands of compromised credentials.

📅 Apr 13, 2026 10:46 AM EDT| Source: The Hacker News ↗
▶Attack Flow
W3LL Phishing-as-a-Service Empire Dismantled by FBI and Indonesian Police
W3LL provided pre-built phishing kits that deployed adversary-in-the-middle (AiTM) reverse proxies between victims and legitimate Microsoft 365 login pages, intercepting both credentials and MFA tokens in real time to hijack authenticated sessions. Subscribers managed campaigns through a centralized dashboard, enabling low-skilled criminals to execute sophisticated business email compromise at scale.

❓Am I Affected?

You are likely at risk if any of the following apply:

⚠Your organization uses Microsoft 365 with SMS, phone call, or push-notification-based MFA (non-phishing-resistant)
⚠Your users regularly click links in emails to authenticate to cloud services
⚠Your environment lacks Conditional Access policies that enforce compliant or managed devices for Microsoft 365 sign-ins
✓Your organization has deployed FIDO2 security keys or passkeys as the primary authentication method for Microsoft 365
✓Your environment enforces token binding and Continuous Access Evaluation (CAE) for all Microsoft 365 sessions
⚠Your organization has no visibility into anomalous sign-in locations, impossible travel, or session cookie reuse in Azure AD logs
🛠Fix & Recommendations
⚠ Immediate 24–72 hours
  • 1Audit your Azure AD / Entra ID sign-in logs for the past 90 days for anomalous patterns: impossible travel, unfamiliar IP addresses, or sign-ins from known proxy/VPN infrastructure immediately following legitimate MFA-authenticated sessions.
  • 2Enable Microsoft Entra ID Conditional Access policies requiring compliant or hybrid Azure AD-joined devices for all Microsoft 365 access, which blocks session tokens stolen via AiTM proxies from being replayed on unmanaged devices.
  • 3Revoke refresh tokens and active sessions for any accounts showing suspicious sign-in activity using 'Revoke-AzureADUserAllRefreshToken' or via the Entra admin center.
  • 4Deploy anti-phishing URL filtering rules to block known AiTM proxy domains and enable Safe Links in Microsoft Defender for Office 365 with URL detonation enabled.
  • 5Issue an organization-wide alert to users warning about phishing emails impersonating Microsoft 365 login pages, emphasizing that legitimate Microsoft sign-ins will never redirect through unfamiliar domains.
● Strategic This week & beyond
  • 1Migrate all users to FIDO2 security keys or platform passkeys as the primary authentication method for Microsoft 365, eliminating the credential and MFA token interception vector that AiTM attacks exploit.
  • 2Implement Continuous Access Evaluation (CAE) and token binding in Entra ID to ensure that stolen session tokens are rapidly invalidated when client context changes (e.g., IP address shift).
  • 3Deploy a phishing-resistant authentication architecture across all cloud services, not just Microsoft 365, including SAML/OIDC-based SSO providers, using hardware-bound credentials.
  • 4Establish automated detection and response playbooks for business email compromise indicators: mailbox rule creation (forwarding, auto-delete), anomalous email volume, and payment-related keyword patterns in outbound mail.
  • 5Conduct regular red-team exercises specifically testing AiTM phishing resilience, including evilginx-style proxy simulations, to validate that Conditional Access and token binding controls are effective.
📚Concept BriefingClick a tag to expand
Phishing-as-a-Service
A criminal business model where ready-made phishing toolkits, infrastructure, and support are sold as subscriptions, allowing low-skilled attackers to launch sophisticated campaigns.
W3LL was a leading PhaaS platform serving 500+ threat actors. Its takedown disrupts a major supply chain, but the model is easily replicated by competitors.
Adversary-in-the-Middle
A technique where a malicious proxy sits between the victim and a legitimate service, relaying login credentials and MFA tokens in real time to hijack authenticated sessions.
AiTM was W3LL's core technical capability, enabling it to defeat standard multi-factor authentication on Microsoft 365 accounts.
Business Email Compromise
A fraud scheme where attackers use compromised or spoofed business email accounts to trick employees, partners, or vendors into transferring funds or sharing sensitive data.
W3LL-enabled credential theft directly fueled BEC campaigns responsible for an estimated $20 million in fraud attempts.
FIDO2 / Passkeys
A phishing-resistant authentication standard using public-key cryptography bound to the legitimate server's domain, making credential interception by proxy mathematically impossible.
FIDO2 is the primary countermeasure against the AiTM technique W3LL exploited — organizations using passkeys would have been immune to these attacks.
Continuous Access Evaluation
A security mechanism where access tokens are re-validated in near real-time based on changes in user context (location, device compliance, risk level), enabling rapid revocation of compromised sessions.
CAE can mitigate AiTM-stolen session tokens by detecting and terminating sessions when the client IP or device context changes unexpectedly.
🔍Threat Hunting Signals
High AiTM session cookie replay from divergent IP ►
Look for Microsoft 365 sign-in events where a user authenticates with MFA from one IP address and then, within minutes, the same session token is used from a completely different IP address or ASN. This pattern indicates a stolen session cookie being replayed by an attacker. Filter Azure AD sign-in logs for successful authentications followed by immediate activity from non-matching IPs.
Azure AD / Entra ID Sign-in LogsMicrosoft 365 Unified Audit LogMicrosoft Defender for Cloud Apps
High Suspicious mailbox rule creation post-authentication ►
Monitor for new inbox rules created shortly after authentication that forward, redirect, or delete emails — particularly rules targeting keywords like 'invoice', 'payment', 'wire', or 'bank'. Attackers create these rules to intercept financial communications and hide their activity from the legitimate mailbox owner.
Microsoft 365 Unified Audit Log (New-InboxRule, Set-InboxRule)Microsoft Defender for Office 365 Alerts
High Authentication to Microsoft 365 via known AiTM proxy infrastructure ►
Correlate sign-in source IPs against threat intelligence feeds for known evilginx, Modlishka, and W3LL proxy infrastructure. AiTM proxies often use hosting providers and ASNs distinct from typical corporate or consumer ISPs. Flag sign-ins from VPS/cloud hosting providers that are atypical for your user population.
Azure AD / Entra ID Sign-in LogsNetwork proxy/firewall logsThreat intelligence platform
Medium Impossible travel or anomalous geo-location patterns ►
Detect sign-in events where the same user account authenticates from geographically distant locations within an impossibly short timeframe. While this is a known detection, in the context of AiTM attacks it specifically indicates real-time session hijacking where the attacker's location differs from the victim's.
Azure AD Identity ProtectionMicrosoft Defender for Cloud AppsSIEM correlation rules
Medium Bulk email access or email forwarding to external addresses ►
Monitor for accounts that begin accessing large volumes of email via Graph API, EWS, or IMAP shortly after authentication, or that configure auto-forwarding to external email addresses. This is a common post-compromise action for BEC preparation.
Microsoft 365 Unified Audit Log (MailItemsAccessed, Set-Mailbox)Microsoft Defender for Cloud Apps
