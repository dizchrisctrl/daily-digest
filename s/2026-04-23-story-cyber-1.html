Chinese APT Weaponizes Outlook, Slack, and Discord as C2 Channels to Spy on Mongolia

High ⚠ Mitigation Only

A Chinese state-sponsored threat actor is leveraging legitimate cloud services — Microsoft Outlook, Slack, Discord, and file.io — as command-and-control infrastructure in an espionage campaign targeting Mongolian government networks. By tunneling C2 traffic through trusted SaaS platforms, the attackers make detection extraordinarily difficult since security tools see legitimate API calls rather than suspicious connections, turning every organization's reliance on cloud collaboration tools into a potential blind spot.

📅 Apr 23, 2026 9:00 PM EDT| Source: darkreading ↗
▶Attack Flow
Chinese APT Abuses Outlook, Slack, Discord, and file.io as C2 Channels Targeting Mongolia
The APT deploys malware that communicates via the APIs of legitimate cloud services — reading commands from Outlook drafts, Slack private channels, and Discord webhooks, and exfiltrating data through file.io's ephemeral file-sharing service. This multi-channel approach provides redundancy and blends malicious traffic with normal enterprise SaaS usage, bypassing network-layer detection.

❓Am I Affected?

You are likely at risk if any of the following apply:

⚠Your environment allows unrestricted outbound HTTPS access to Outlook (outlook.office365.com), Slack (slack.com), Discord (discord.com), and file.io without API-level inspection
⚠Your organization operates in a government, diplomatic, or geopolitically sensitive sector in the Asia-Pacific region
⚠Your environment permits unmanaged or loosely governed OAuth application registrations in Microsoft 365 / Azure AD
⚠Your endpoint detection and response (EDR) solution does not monitor API-level interactions with cloud services from endpoint processes
✓Your organization blocks Discord, file.io, and non-sanctioned Slack workspaces at the network boundary and monitors Microsoft Graph API usage via CASB
✓Your environment has no Microsoft 365 deployment and does not permit Slack or Discord usage on corporate endpoints
🛠Fix & Recommendations
⚠ Immediate 24–72 hours
  • 1Block outbound connections to file.io (file.io, *.file.io) at the proxy/firewall immediately — this service has minimal legitimate enterprise use and is a known exfiltration vector.
  • 2If Discord is not business-sanctioned, block discord.com and cdn.discordapp.com at the proxy/firewall; if sanctioned, restrict webhook creation and monitor outbound POST requests to Discord webhook URLs from non-browser processes.
  • 3Audit all OAuth app registrations and consent grants in Azure AD / Entra ID for Microsoft Graph API permissions (especially Mail.ReadWrite, Mail.Send, Files.ReadWrite.All) — revoke any unrecognized or overprivileged apps immediately using: Get-AzureADServicePrincipal | Where-Object {$_.Tags -notcontains 'WindowsAzureActiveDirectoryIntegratedApp'}.
  • 4Deploy endpoint detection rules to alert on non-standard processes (i.e., not outlook.exe, slack.exe, or browser binaries) making API calls to graph.microsoft.com, slack.com/api, or discord.com/api endpoints.
  • 5Hunt for anomalous Outlook draft activity: query Microsoft 365 Unified Audit Logs for high-frequency MailItemsAccessed or Update operations on Draft folders from unusual client IDs or IP ranges.
● Strategic This week & beyond
  • 1Deploy a Cloud Access Security Broker (CASB) with inline API inspection to gain visibility into Microsoft Graph API, Slack API, and Discord API traffic — enable anomaly detection for unusual volumes, timing, or client application patterns.
  • 2Implement conditional access policies in Azure AD / Entra ID that restrict Microsoft Graph API access to managed devices and approved applications only, using app-enforced restrictions and compliant device requirements.
  • 3Adopt behavioral analytics (UEBA) on endpoint telemetry to detect processes that mimic legitimate SaaS client behavior but originate from unsigned binaries, unusual file paths, or exhibit C2-like polling patterns (periodic beaconing intervals).
  • 4Segment and monitor internal networks so that only designated service accounts and endpoints can reach SaaS API endpoints — enforce this via micro-segmentation or identity-aware proxies rather than simple allowlisting.
  • 5Establish a SaaS governance program that inventories all sanctioned cloud services, restricts OAuth consent to admin-approved apps, and continuously monitors for shadow IT usage of collaboration platforms that could serve as C2 channels.
📚Concept BriefingClick a tag to expand
Living-off-Trusted-Services
An attack technique where adversaries route malicious command-and-control traffic through legitimate, widely-trusted cloud services instead of attacker-owned infrastructure.
This Chinese APT uses Outlook, Slack, Discord, and file.io as C2 relays, making malicious traffic blend with normal enterprise SaaS usage and rendering traditional network-based detection ineffective.
Multi-Channel C2 Redundancy
An adversary tactic of maintaining multiple independent command-and-control channels simultaneously so that blocking one does not sever the attacker's access.
The APT operates across at least four cloud services at once, ensuring that if Mongolian defenders block one channel, the implant seamlessly falls back to another.
API-Level Abuse
Exploiting the programmatic interfaces (APIs) of legitimate services for unauthorized purposes, such as reading commands from draft emails or posting data via webhooks.
The attackers leverage Microsoft Graph API for Outlook draft manipulation and Discord webhooks for payload delivery, activities that appear identical to legitimate API usage.
Ephemeral Exfiltration Staging
Using temporary or self-destructing file-sharing services to stage stolen data, minimizing the forensic footprint available to investigators.
file.io's auto-delete-after-first-download model is used by the APT to exfiltrate data while leaving minimal evidence for incident responders.
SaaS Blind Spot
A gap in security visibility that arises because traffic to trusted SaaS platforms is typically allowlisted and encrypted, preventing inspection by traditional security tools.
The campaign exploits the fundamental trust organizations place in cloud collaboration tools — firewalls, proxies, and DLP see only TLS connections to trusted domains and cannot distinguish C2 traffic from legitimate use.
🔍Threat Hunting Signals
High Non-standard processes accessing Microsoft Graph API ►
Look for processes other than Outlook, Teams, OneDrive, or known browsers making HTTPS connections to graph.microsoft.com or outlook.office365.com. Malware implants will use custom binaries or injected processes to call Graph API endpoints for reading/writing Outlook drafts. Check for unsigned executables or processes running from unusual directories (e.g., %TEMP%, %APPDATA%\unusual_folder) establishing connections to these domains.
EDR telemetry (process-to-network mapping)Proxy/firewall logsSysmon Event ID 3 (Network Connection)
High Anomalous Outlook Draft folder activity ►
Query Microsoft 365 Unified Audit Logs for high-frequency MailItemsAccessed or Update operations targeting the Drafts folder, especially from non-interactive or application-context sessions, unusual OAuth app client IDs, or IP addresses outside expected ranges. C2 via drafts typically creates a pattern of rapid create-read-delete cycles on draft messages.

High Outbound Discord webhook POST requests from non-browser processes ►
Monitor for HTTP POST requests to discord.com/api/webhooks/* originating from processes that are not the Discord desktop client or a standard browser. This pattern indicates a malware implant using Discord webhooks as a C2 or exfiltration channel. Pay attention to periodic, fixed-interval POST requests which suggest automated beaconing.

Medium Connections to file.io from enterprise endpoints ►
Any connection to file.io or its associated CDN domains from corporate endpoints should be investigated as potential exfiltration staging. This ephemeral file-sharing service has very limited legitimate enterprise use. Look for POST requests (file uploads) followed by GET requests to unique download URLs.

Medium Slack API calls from unauthorized applications or tokens ►
Review Slack audit logs (Enterprise Grid) or workspace access logs for API calls originating from unrecognized app tokens, bot tokens, or OAuth scopes not associated with sanctioned integrations. Look for messages being posted to or read from private channels by tokens that do not correspond to known Slack apps in the workspace.
