CISA Adds Windows Task Host Privilege Escalation to Known Exploited Vulnerabilities List

CVSS
7.8
Critical ✓ Patch Available

CISA has added a Windows Task Host privilege escalation vulnerability to its Known Exploited Vulnerabilities (KEV) catalog, confirming active exploitation in the wild. Attackers with local access to a Windows machine can exploit a flaw in taskhostw.exe to escalate from a standard user account to SYSTEM-level privileges, granting full administrative control. All organizations running Windows 10, Windows 11, and Windows Server should treat this as an urgent patching priority.

📅 Apr 15, 2026 10:51 AM EDT| Source: BleepingComputer ↗
▶Attack Flow
CVE-2026-XXXXX – Windows Task Host (taskhostw.exe) Privilege Escalation (CISA KEV)
A local attacker exploits a logic flaw in the Windows Task Host process (taskhostw.exe) to hijack its SYSTEM-level security token, escalating privileges from a standard user to full SYSTEM access without requiring additional credentials or user interaction.

❓Am I Affected?

You are likely at risk if any of the following apply:

⚠You run any version of Windows 10, Windows 11, or Windows Server that has not applied the April 2026 cumulative security update.
⚠Your environment includes Windows endpoints where standard (non-admin) users have interactive or remote logon access.
⚠You are a U.S. federal agency subject to BOD 22-01 and have not yet applied the patch within the CISA-mandated timeline.
⚠Your environment has endpoint detection and response (EDR) solutions actively monitoring for process injection and token manipulation on all Windows hosts.
✓You have already applied the April 2026 cumulative update to all Windows systems and verified successful installation.
✓Your environment exclusively runs non-Windows operating systems (Linux, macOS) with no Windows endpoints or servers.
🛠Fix & Recommendations
⚠ Immediate 24–72 hours
  • 1Apply the April 2026 Microsoft cumulative security update to all Windows 10, Windows 11, and Windows Server systems immediately. Use WSUS, SCCM/MECM, Intune, or manual Windows Update to push patches. Prioritize internet-facing and high-value systems first.
  • 2Verify patch installation across your fleet: run 'Get-HotFix | Sort-Object InstalledOn -Descending | Select-Object -First 10' on endpoints or check compliance dashboards in your patch management tool.
  • 3Audit scheduled tasks on critical systems for anomalies: run 'schtasks /query /fo LIST /v' and review tasks running under SYSTEM context that were recently created or modified by non-admin accounts.
  • 4Enable enhanced Windows Security Event logging (Event IDs 4688, 4672, 4673) and Sysmon process creation logging to detect any suspicious activity involving taskhostw.exe before the patch is fully deployed.
  • 5If patching cannot be completed within 24 hours, restrict interactive and remote logon rights for non-essential user accounts on critical servers using Group Policy (Computer Configuration > Windows Settings > Security Settings > Local Policies > User Rights Assignment).
● Strategic This week & beyond
  • 1Implement a least-privilege access model across all Windows environments: remove unnecessary local admin rights, enforce tiered administration, and deploy Microsoft LAPS or a PAM solution for just-in-time privilege elevation.
  • 2Deploy and tune EDR/XDR solutions to detect privilege escalation patterns including SYSTEM token impersonation, unusual child processes spawned by taskhostw.exe, and anomalous scheduled task creation.
  • 3Establish an accelerated patch cycle for CISA KEV-listed vulnerabilities with a target SLA of 48 hours for critical actively-exploited flaws, supported by automated patch testing and deployment pipelines.
  • 4Harden Windows Task Scheduler attack surface: audit and restrict who can create or modify scheduled tasks via Group Policy, remove unnecessary tasks, and monitor changes with Windows Event Forwarding (WEF).
  • 5Adopt a defense-in-depth strategy that assumes privilege escalation will occur: segment networks, enforce credential hygiene (Credential Guard, restricted admin mode for RDP), and implement detection for post-exploitation behaviors like credential dumping and lateral movement.
📚Concept BriefingClick a tag to expand
Privilege Escalation
An attack technique where an adversary with limited access exploits a vulnerability to gain higher-level permissions, such as moving from a standard user account to full administrative (SYSTEM) control.
This vulnerability allows attackers who already have a foothold on a Windows machine to escalate to SYSTEM privileges via the Task Host process, turning a minor compromise into full system control.
CISA KEV Catalog
A curated list maintained by the U.S. Cybersecurity and Infrastructure Security Agency of vulnerabilities confirmed to be actively exploited in real-world attacks, triggering mandatory patching deadlines for federal agencies under Binding Operational Directive 22-01.
CISA's addition of this Task Host flaw to the KEV catalog confirms it is being exploited in the wild and creates an enforceable patching deadline for federal agencies, while signaling urgency for all organizations.
Windows SYSTEM Privileges
The highest level of access on a Windows operating system, granting unrestricted control over all system resources, processes, files, and registry settings — more powerful than a standard administrator account.
Exploiting this vulnerability grants attackers SYSTEM-level access, enabling them to install persistent backdoors, dump all credentials, disable security tools, and move laterally across the network.
taskhostw.exe
A core Windows system process (Task Host Window) responsible for executing DLL-based scheduled tasks and COM handler triggers, running under the SYSTEM account by default.
This process is the vulnerable component — attackers exploit a flaw in how it validates or handles inputs to hijack its elevated SYSTEM token and execute arbitrary code.
Defense in Depth
A security strategy that layers multiple independent defensive controls so that if one layer fails, others still protect the environment — rather than relying on a single point of prevention.
This exploit chain shows why preventing initial access alone is insufficient; organizations need layered defenses including patching, EDR, least privilege, and network segmentation to contain privilege escalation attacks.
🔍Threat Hunting Signals
High Anomalous child processes spawned by taskhostw.exe ►
Look for taskhostw.exe spawning unexpected child processes such as cmd.exe, powershell.exe, whoami.exe, net.exe, or other reconnaissance/post-exploitation binaries. Legitimate taskhostw.exe behavior involves running specific scheduled task DLLs, not spawning interactive shells or system utilities. Any deviation is a strong indicator of exploitation.
Windows Security Event Log (Event ID 4688 with command-line auditing)Sysmon (Event ID 1 - Process Creation)EDR telemetry
High SYSTEM token assignment to unexpected processes ►
Monitor for Event ID 4672 (Special Privileges Assigned to New Logon) where the target process or user context is unusual — particularly when a process tree originates from taskhostw.exe and a non-service account suddenly receives SeAssignPrimaryTokenPrivilege or SeDebugPrivilege. This may indicate token manipulation following exploitation.
Windows Security Event Log (Event ID 4672)Windows Security Event Log (Event ID 4673)EDR telemetry
High Suspicious scheduled task creation or modification ►
Hunt for newly created or modified scheduled tasks (Event ID 4698/4702) that run under SYSTEM context, were created by non-administrative user accounts, or reference unusual executable paths or DLLs. Attackers may create or modify tasks as part of the exploitation chain or to establish persistence after escalation.
Windows Security Event Log (Event IDs 4698, 4702)Microsoft-Windows-TaskScheduler/Operational logSysmon (Event ID 11 - File Create for task XML files in C:\Windows\System32\Tasks)
Medium Credential dumping activity following taskhostw.exe exploitation ►
After gaining SYSTEM privileges, attackers commonly target LSASS for credential harvesting. Look for processes accessing lsass.exe memory (Sysmon Event ID 10 with GrantedAccess values like 0x1010 or 0x1FFFFF), or the creation of memory dump files. Correlate with any prior anomalous taskhostw.exe activity in the same timeframe.
Sysmon (Event ID 10 - Process Access targeting lsass.exe)EDR telemetryWindows Defender Credential Guard alerts
Medium Unusual DLL loading by taskhostw.exe ►
Monitor for taskhostw.exe loading DLLs from non-standard paths (outside C:\Windows\System32 or known task directories). Attackers may use DLL side-loading or injection to execute malicious code within the context of the trusted Task Host process. Sysmon ImageLoad events filtered for taskhostw.exe with unusual ImageLoaded paths are key indicators.
Sysmon (Event ID 7 - Image Loaded)EDR DLL load telemetry
